scooby

scooby is a serverless data store intended for lightweight, temporary data storage. Absolutely no configuration or authentication is required. Client applications can use the simple web api to store and retrieve small amounts of JSON.

scooby is very limited in its functionality. Only two endpoints are exposed.

POST https://datascooby.azurewebsites.net/api/values
GET https://datascooby.azurewebsites.net/api/values?obj=

here is how you can put data in

Simply post your JSON to the endpoint. scooby will remember the different kinds of objects you post.

<script src="https://code.jquery.com/jquery-3.5.1.min.js"

<script>

    var data = { name: "the dude", age: 39 };

    $.ajax({
        type: "POST",
        contentType: "application/json; charset=utf-8",
        url: "https://datascooby.azurewebsites.net/api/values",
        dataType: "json",
        data: JSON.stringify(data),
        success: function (response) {

        }
    });
</script>

here is how you can get data out

To get your objects, simply issue a GET request to the endpoint, passing in the JSON schema so scooby knows what kind of objects to get.

Caveats

Each client application is limited to storing 250KB of JSON
